Notes

The data argument is a set of lists with length equal to the number of systems. These data are unpacked and vertically concatenated in this class. The parameter self._data will have length equal to the sum of the number of time gates in each system. The same is true for the errors, and the predicted data vector.

updateErrors(relativeErr, additiveErr)Â¶

Updates the data errors

Assumes a t^-0.5 behaviour e.g. logarithmic gate averaging V0 is assumed to be ln(Error @ 1ms)

Parameters

  • relativeErr (list of scalars or list of array_like) â A fraction percentage that is multiplied by the observed data. The list should have length equal to the number of systems. The entries in each item can be scalar or array_like.

  • additiveErr (list of scalars or list of array_like) â An absolute value of additive error. The list should have length equal to the number of systems. The entries in each item can be scalar or array_like.

Raises

  • TypeError â If relativeErr or additiveErr is not a list

  • TypeError â If the length of relativeErr or additiveErr is not equal to the number of systems

  • TypeError â If any item in the relativeErr or additiveErr lists is not a scalar or array_like of length equal to the number of time channels

  • ValueError â If any relative or additive errors are <= 0.0
